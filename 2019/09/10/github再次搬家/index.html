github再次搬家

发表于 2019-09-10

做笔记或者做记录 的好处 我是体验到了！ 当再次遇到相同的问题的时候，我们 可以很快的并且心里很有谱的将它解决，比如 今天我的github本地环境再次搬家！ 哭啊！

前景提要：

今天下午我们没有课，想借助这个时间多学些知识呢，刚打开电脑不到三分钟，不知为何突然灭屏再打不开机了（几乎是）
让戴尔售后人员看了下，最后确定为系统文件出错了，直接恢复了出厂设置，哈，够狠！我猜测是我昨天在做逆向题时 修改了系统文件（但仔细想 修改应该不是主要文件啊）
但保险起见以后都在虚拟机里搞吧，这太吓人了！

首先
我的绝大部分文件都没有放到C盘，所以恢复出厂设置后除了时间浪费之外对我没有多大影响（时间明明就有很大影响好吧！），主要是Github本地环境 与 服务器断了连接！这里简单记录下 再次建立连接的过程

新建一个我们的Github本地环境（文件夹）

我再F盘建了个文件夹，把原来在D盘的文件夹放在了这里面

然后 将GIT和nodejs都安装到了这里（实现集中化）

配置git

安装完Git要设置用户名称和邮件地址。这样做很重要，
因为每一个Git的提交都会使用这些信息，并且它会写入你的每一次提交中，不可更改：
$ git config –global user.name “yangmutou”
$ git config –global user.email “1594783824@qq.com“

然后 执行命令 git config -l

这里 贴张图：

与Github 建立 联系- 设置 ssh keys

ssh-keygen -t rsa -C “1594783824@qq.com“

然后 C:\Users\杨木头.ssh 文件夹里就会 产生 2个文件

我们把id_rsa.pub中的 公钥 给复制下来
然后在 Github 进入 这个页面 -NewSSH

好了（我想上面那俩，已经没有用了 不过暂时留着 做纪念）

下面 我们测试下 与github建立连接没！
ssh -T git@github.com
提示：

OK，我们 不管他 输入 yes！

和我上次 连接成功 不一样 上次是直接 （下图）

继续往下走

模块安装

将这些关键文件 放在新的文件夹里 yangmutou’blog中

在这个文件夹里右键 选择 git bash
输入 npm install 命令

不用管 ，都是警告罢了，不影响我们使用
然后依次执行以下三种指令
$ npm install hexo-deployer-git –save
$ npm install hexo-generator-feed –save
$ npm install hexo-generator-sitemap –save

最后
$ hexo generate
$ hexo deploy
然后弹出
Github登录页面
账号密码 填入 就可以了

登录后没反应
其实是成功了

就能在 http://localhost:4000 本地 浏览了

正常使用了
