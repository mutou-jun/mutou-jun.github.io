0x01 re1

F5 反编译

if 我们输入的v5等于v9（说明 flag字符串已经保存在内存中） 便输出flag

可F12却不能找到找到 flag 这个字符串

查阅资料了解到，反编译的结果不是一定正确的，IDA采用递归下降法进行反编译，它的优点在于很少会在反编译时把数据当作代码来处理，不过这次IDA很明显把flag当成代码，进行了反编译，因此在string界面无法找到flag

解决方法

  1. 是在IDA打开文件时选择binaryfile,在此模式下IDA不会进行反编译，此时Shift +F12再打开strings界面就可发现flag
  2. 在HXD 16进制编辑软件 Cral+F 搜索DTUCTF
  3. 在IDAHex View-1窗口 seacher 寻找DTUCTF
